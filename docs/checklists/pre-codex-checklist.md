# Pre-Codex Checklist

Run this checklist before asking Codex to implement.

- [ ] Requirement is clear (goal, in-scope, non-scope).
- [ ] `docs/product-scope.md` is up to date.
- [ ] Relevant spec files are created/updated.
- [ ] Acceptance criteria are explicit and testable.
- [ ] Constraints are listed (runtime, secret handling, deployment).
- [ ] Task requests minimal diff and forbids unrelated refactor.
- [ ] Files expected to change are listed.
- [ ] PR/review expectations are defined.
